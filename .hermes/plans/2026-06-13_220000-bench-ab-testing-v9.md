## Анализ провалов по категориям

### TERMINAL (2 провала)

**TERM-007** (рефакторинг кода) — 4/6 баллов
- Проблема: `temp` всё ещё присутствует в коде. Модель заменяет только некоторые вхождения `temp` → `result`, но пропускает случаи вроде `let temp2 = process(temp)` где `temp` используется как аргумент функции.
- Корень: regex-замена модели неточна. Нужно явно указать "замени ВСЕ вхождения temp включая как часть других слов" или использовать `runCommand` с `sed`.
- Гипотеза: добавить в промпт пример с заменой переменных через `runCommand` + `sed`, или явно указать "заменить temp везде, включая аргументы функций".

**TERM-008** (слияние JSON) — 0/2 баллов, 20 шагов, таймаут
- Проблема: модель читает 3 файла, но не может объединить массивы. 20 шагов — упёрлась в лимит.
- Корень: задача требует прочитать несколько JSON, объединить массивы, отсортировать. Модель не может держать в контексте данные из 3 файлов одновременно (truncation 800 chars).
- Гипотеза: увеличить лимит обрезки с 800 до 2000 символов. Или добавить в промпт явный паттерн "прочитай все файлы, потом напиши один writeFile с объединёнными данными".

## План A/B тестирования

### Методология
1. Каждая гипотеза тестируется ОТДЕЛЬНО на одном и том же наборе задач
2. Бенчмарк запускается на провалившихся задачах + 5 случайных прошедших (для регрессии)
3. Сравниваем: score, steps, toolCalls, duration
4. Если гипотеза улучшает результат без регрессии — принимаем
5. Итерации до достижения 80%+

### Порядок тестирования

#### Раунд 1: P0 гипотезы (ожидание: 73% → 78-83%)
- v9a: Г1 (обрезка 2000)
- v9b: Г2 (ранняя подсказка read→write)
- v9c: Г3 (чеклист markdown)
- v9d: Г1 + Г2 + Г3 (комбо)

#### Раунд 2: P1 гипотезы (ожидание: 78-83% → 82-88%)
- v9e: Г4 (пример sed)
- v9f: Г5 (усиленный createPlan)
- v9g: Г6 (изменённый eval)

#### Раунд 3: P2 гипотезы (ожидание: 82-88% → 85-92%)
- v9h: Г7 (retry при 0 шагах)
- v9i: Г8 (cooldown 3s)

#### Раунд 4: Комбо лучших
- v9-final: все успешные гипотезы вместе

### Критерии остановки
- Достижение 85%+ общего скора
- Исчерпание гипотез
- 3 итерации подряд без улучшения

## Файлы для изменения

1. **lib.ts** — BENCH_SYSTEM_PROMPT, runReActLoop (подсказки), parseAction, обрезка
2. **bench/runner.ts** — executeTask (retry, cooldown), taskTimeout
3. **bench/tasks/tool-use.ts** — eval для TOOL-001, TOOL-005, TOOL-007
4. **bench/tasks/terminal.ts** — eval для TERM-007 (опционально)

## Риски

1. **Переоптимизация под бенчмарк** — изменения eval (Г6) могут не отражать реальное улучшение агента
2. **Увеличение времени бенчмарка** — retry и cooldown увеличивают время
3. **Конфликт гипотез** — комбо может дать худший результат чем отдельные изменения
4. **Нестабильность LM Studio** — 0 шагов может быть проблемой инфраструктуры, не агента
