# План: Ретро/Киберпанк TUI для tiny-agent

## Цель

Доработать консольный интерфейс tiny-agent в стиле ретро/киберпанк, используя принципы из скилла tui-designer. Сохранить скриншоты/артефакты для обсуждения.

## Текущий контекст

- **Проект**: `/home/serg/tiny-agent` — консольный LLM-агент на Node.js/TypeScript
- **Текущий TUI**: `agent.ts` — чистый `readline/promises` + ANSI escape-коды (8 цветов)
- **Цвета**: базовые ANSI (red, green, blue, cyan, magenta, yellow) + bright/dim
- **Статус-строка**: `[model]>[mode:CWD]>[git:branch]>[context]`
- **Зависимости**: минимум (cheerio, playwright), пользователь предпочитает zero-dependency core
- **Стиль кода**: компактный, без тяжёлых фреймворков

## Что можно сделать (без новых зависимостей)

Консольный Node.js ограничен — настоящие CRT-эффекты (scanlines, glow, curvature) требуют веб-рендеринга. Но в рамках чистого ANSI можно:

### Уровень 1: Улучшенный ANSI-дизайн (zero-dependency)

1. **Box-drawing рамки** — заменить `─`.repeat(50) на полноценные рамки из Unicode box-drawing characters (`╔═╗║╚═╝╠╣╦╩╬`)
2. **Цветовая палитра** — перейти на 256-color ANSI для более насыщенных оттенков (phosphor green `#00ff00`, cyberpunk cyan `#00ffff`, magenta `#ff00ff`)
3. **Секции с заголовками** — структурированный вывод: SYSTEM STATUS, TOOL OUTPUT, AGENT RESPONSE с рамками
4. **Прогресс-бар** — для шагов агента: `[████████░░] 8/10`
5. **Модальный индикатор** — визуально выделить режим PLAN / WRITE
6. **Статус-бар** — переработать с использованием box-drawing и цветов
7. **Анимация курсора** — мигающий блок `█` вместо простого `>`

### Уровень 2: Веб-рендерер с CRT-эффектами (отдельный модуль)

Создать отдельный файл `src/tui-renderer.ts`, который генерирует HTML-страницу с:
- CSS scanlines overlay
- Neon glow текст
- CRT curvature эффект
- Phosphor green / cyberpunk палитра
- Автообновление через WebSocket или polling

Это не заменит консольный TUI, а будет дополнительным режимом `--web`.

## Box-drawing символы

```
Горизонтальные: ─ (light), ━ (heavy), ═ (double)
Вертикальные:   │ (light), ┃ (heavy), ║ (double)
Углы:           ┌ ┐ └ ┘ (light), ┏ ┓ ┗ ┛ (heavy), ╔ ╗ ╚ ╝ (double)
Т-образные:     ├ ┤ ┬ ┴ ┼ (light)
Скруглённые:    ╭ ╮ ╰ ╯
```

## Риск и ограничения

- **Терминальная совместимость**: box-drawing символы могут не работать в старых терминалах (PowerShell < 7, некоторые SSH). Нужен fallback на ASCII.
- **256-color**: не все терминалы поддерживают. Нужен fallback на 16-color.
- **Ширина терминана**: фиксированная ширина рамок может ломаться на узких терминалах. Нужна адаптация под `process.stdout.columns`.
- **Производительность**: ANSI-коды дешёвые, но частый рефреш может мерцать.
