# План: Починка RLM бенчмарка

## Цель
Исправить RLM задачи чтобы они реально тестировали rlm инструмент и проходили стабильно.

## Текущее состояние

Последний запуск (6 задач, RLM-003 удалён):

| Задача | Результат | Проблема |
|--------|-----------|----------|
| RLM-001 | ✅ PASS 100% | OK |
| RLM-002 | ❌ FAIL 67% | result.txt не создал — агент не понимает что factorial(5)=120 |
| RLM-004 | ❌ FAIL 86% | draft == final (rlm вызван 1 раз, нужно 2 с разными промптами) |
| RLM-005 | ❌ таймаут 600s | завис на writeFile с длинным content |
| RLM-006 | ❌ не запускался | LM Studio не отвечает после RLM-005 |
| RLM-007 | ❌ не запускался | LM Studio не отвечает |

Корневая проблема: **LM Studio нестабилен** — после 2-3 rlm вызовов начинает тормозить/выгружать модель.

## План действий

### Шаг 1: Уменьшить нагрузку на LM Studio
- Уменьшить maxSteps для hard/expert с 25/20 до 12/10
- Уменьшить таймаут rlm с 120s до 60s (быстрее fail = быстрее следующая задача)
- Увеличить cooldown между задачами до 5s

### Шаг 2: Переписать RLM-002
- Промпт должен явно говорить "запиши 120 в result.txt"
- Или использовать rlm для вычисления факториала и записи результата

### Шаг 3: Переписать RLM-004
- Два разных rlm вызова: первый summarize, второй improve
- Явно указать что draft и final должны быть разными

### Шаг 4: Переписать RLM-005
- Упростить промпт — меньше шагов
- refactor_report.txt должен создаваться агентом, не копировать content из rlm

### Шаг 5: Переписать RLM-006 и RLM-007
- Уменьшить количество rlm вызовов до 1 на задачу
- Упростить промпты

### Шаг 6: Запустить и проверить
- Запустить RLM бенчмарк
- Проверить что все задачи завершаются
- Сохранить отчёт

## Файлы для изменения
- `bench/tasks/rlm.ts` — промпты и evaluate для RLM-002, 004, 005, 006, 007
