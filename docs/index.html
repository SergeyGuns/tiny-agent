tiny-agent

v0.1.1 — Autonomous AI Coding Agent

Минималистичный фреймворк для создания автономных ИИ-разработчиков на TypeScript. Чистый RLM-протокол, ноль тяжёлых абстракций, полная прозрачность.

Установить npm GitHub
Установка
npm install -g @sergey-guns/tiny-agent
Алиасы: tia · ty · ta

Возможности

🧠

RLM Architecture

Recursive Language Model — модель сама решает, сколько вызовов инструментов сделать за один шаг. Результаты агрегируются как батч.

⚡

Multi-Action

Параллельные независимые вызовы инструментов за один ответ: чтение нескольких файлов, поиск + запись, и т.д.

🔒

Zero-Dependency Core

Только нативный fetch и стандартная библиотека Node.js. Никаких LangChain, LlamaIndex и других абстракций.

📡

Любой провайдер

LM Studio, OpenRouter, Anthropic, OpenAI — единый интерфейс. Переключение через \provider в TUI.

🔄

READ ≠ WRITE

Агент не записывает файлы, если его просто попросили прочитать или объяснить. Защита от циклов и саморекурсии.

📎

@filename TAB

Автодополнение имён файлов в TUI с прикреплением содержимого. Работает с поддиректориями.

Быстрый старт

# Установка npm install -g @sergey-guns/tiny-agent # Интерактивный TUI tia # Автономный режим tia "Создай CLI утилиту на Node.js для подсчёта строк кода"

Создайте .env в рабочей директории:

# LM Studio (локальная модель) PROVIDER_URL=http://192.168.0.156:1234/v1 MODEL_NAME=qwen/qwen3.5-9b # Или любой другой OpenAI-совместимый API

TUI Команды

\planрежим планирования
\writeрежим записи
\stepsистория шагов
\expand Nразвернуть шаг N
\providerуправление моделями
\testтестовый вывод
\exitвыход
@fileприкрепить файл (TAB)

Инструменты

search_web

Поиск в интернете (DuckDuckGo / Wikipedia)

fetch_url_content

Извлечение текста из URL

list_directory

Просмотр структуры файлов

read_file_content

Чтение исходного кода

write_file_content

Безопасная запись файлов

execute_shell_command

Выполнение команд в терминале

query_language_model

Рекурсивный вызов LLM для анализа

decompose_and_execute

Декомпозиция сложных задач
