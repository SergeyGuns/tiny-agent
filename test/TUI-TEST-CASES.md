# TUI Test Cases — Ручное тестирование

**Инструкция:** запускай каждый кейс через `echo '<промпт>' | node dist/agent.js` и проверяй вывод.

---

## CHAT — Простой разговор (0 инструментов, 0 файлов)

### CHAT-001: Приветствие
- **Prompt:** `Привет`
- **Ожидание:**
  - ✅ Нет вызовов инструментов
  - ✅ Нет созданных файлов
  - ✅ Вывод: `🤖 ASSISTANT: Привет! Чем могу помочь?`
  - ✅ `1 steps  │  tools: 0`

### CHAT-002: Вопрос о возможностях
- **Prompt:** `Что ты умеешь?`
- **Ожидание:**
  - ✅ Ответ текстом (не файл)
  - ✅ Краткое описание возможностей

### CHAT-003: Как дела?
- **Prompt:** `Как дела?`
- **Ожидание:**
  - ✅ Простой дружеский ответ
  - ✅ Без tool calls

---

## READ — Чтение файлов (без записи)

### READ-001: Посмотреть содержимое директории
- **Prompt:** `Посмотри что в текущей директории`
- **Ожидание:**
  - ✅ Вызван `list_directory`
  - ✅ Нет `write_file_content`
  - ✅ Ответ текстом со списком файлов

### READ-002: Прочитать package.json
- **Prompt:** `Прочитай package.json и скажи какие скрипты есть`
- **Ожидание:**
  - ✅ Вызван `read_file_content`
  - ✅ Ответ текстом (без создания файла `summary.md`)

### READ-003: Посмотреть как запускается проект
- **Prompt:** `Посмотри как запускается проект`
- **Ожидание:**
  - ✅ Читает `package.json`, `README.md`
  - ✅ Нет создания `STARTUP_INSTRUCTIONS.md`
  - ✅ Компактный ответ (какие команды доступны)

### READ-004: Найти все TypeScript файлы
- **Prompt:** `Найди все .ts файлы в проекте`
- **Ожидание:**
  - ✅ `list_directory` или `search_in_files`
  - ✅ Список файлов в ответе

---

## WRITE — Запись файлов

### WRITE-001: Создать простой файл
- **Prompt:** `Создай файл hello.txt с текстом Hello World`
- **Ожидание:**
  - ✅ Вызван `write_file_content`
  - ✅ `→ Файл записан: /path/to/hello.txt`
  - ✅ `result: 1 file(s) written(hello.txt)`

### WRITE-002: Создать JSON файл
- **Prompt:** `Создай файл config.json с содержимым {"name": "test", "version": "1.0"}`
- **Ожидание:**
  - ✅ Файл создан с корректным JSON

### WRITE-003: Прочитать и создать отчёт
- **Prompt:** `Прочитай package.json и создай файл summary.md с описанием проекта`
- **Ожидание:**
  - ✅ `read_file_content` → `write_file_content`
  - ✅ `summary.md` содержит описание

### WRITE-004: Создать несколько файлов
- **Prompt:** `Создай два файла: a.txt с текстом "File A" и b.txt с текстом "File B"`
- **Ожидание:**
  - ✅ Два вызова `write_file_content`
  - ✅ Два файла созданы

## MULTI — Многошаговые задачи

### MULTI-001: Прочитать и переформатировать
- **Prompt:** `Прочитай README.md, извлеки все заголовки и сохрани их в файл headings.txt, каждый с новой строки`
- **Ожидание:**
  - ✅ `read_file_content` → `write_file_content`
  - ✅ `headings.txt` содержит заголовки

### MULTI-002: Поиск и запись результата
- **Prompt:** `Найди в проекте все файлы с экспортом функций (export function) и сохрани список в exports.txt`
- **Ожидание:**
  - ✅ `search_in_files` → `write_file_content`
  - ✅ `exports.txt` содержит список

### MULTI-003: Анализ и отчёт
- **Prompt:** `Посмотри все .ts файлы в src/, подсчитай сколько строк кода в каждом и создай отчёт report.md с таблицей: файл | строки`
- **Ожидание:**
  - ✅ Читает несколько файлов
  - ✅ Создаёт `report.md` с таблицей

## EDGE — Граничные случаи

### EDGE-001: Очень длинный промпт
- **Prompt:** `Привет. ` + `Как дела? `.repeat(50)
- **Ожидание:**
  - ✅ Не зацикливается
  - ✅ Ответ за разумное время

### EDGE-002: Запрос на несуществующий файл
- **Prompt:** `Прочитай файл nonexistent.txt`
- **Ожидание:**
  - ✅ Ошибка чтения (не падает)
  - ✅ Сообщение типа `read_file_content error: ...`

### EDGE-004: Противоречивый запрос
- **Prompt:** `Создай файл output.txt но не создавай никаких файлов`
- **Ожидание:**
  - ✅ Не создаёт файл (или создёт, но предупреждает)

## Чек-лист TUI

- [ ]  `👤 You:` отображается перед сообщением пользователя
- [ ]  `🤖 ASSISTANT:` отображается с полным текстом (с переносом строк)
- [ ]  `✓ COMPLETE` показывает статистику (steps, time, tools, result)
- [ ]  Цвета соответствуют неоновой палитре (cyan/magenta/green)
- [ ]  Псевдографика (┏━┓) отображается корректно
- [ ]  Прогресс-бар отображается в автономном режиме
- [ ]  Нет мусора в выводе (лишние `Action:`, пустые строки)

## Быстрый запуск всех кейсов

```bash
# CHAT
echo 'Привет' | node dist/agent.js
echo 'Что ты умеешь?' | node dist/agent.js

# READ
echo 'Посмотри что в текущей директории' | node dist/agent.js
echo 'Прочитай package.json и скажи какие скрипты есть' | node dist/agent.js

# WRITE
echo 'Создай файл hello.txt с текстом Hello World' | node dist/agent.js
echo 'Создай файл config.json с содержимым {"name": "test"}' | node dist/agent.js

# MULTI
echo 'Прочитай README.md, извлеки заголовки и сохрани в headings.txt' | node dist/agent.js
```
